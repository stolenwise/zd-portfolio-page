The core engine

The 7-Step
Response Framework

Every ticket — whether new or a follow-up — runs through this loop. The agent doesn't just pattern-match; it builds a compounding mental model that improves with every exchange.

1
Foundation

Load — 750-Ticket Context

Before reading a single word of the incoming ticket, the agent loads the full historical baseline: your 250 personal tickets plus the broader 500-ticket corpus. This is not a lookup — it is deep contextual fluency.

2
Comprehension

Understand — Patterns, Methodology, Tone

The agent doesn't just read tickets — it understands them. What questions do you ask? In what order? When do you escalate versus keep diagnosing? What vocabulary do you use? What is your average response length? This step extracts the how.

3
Mapping

Apply — Match to Historical Tickets

The current ticket is mapped to the 1–3 most similar historical tickets. It extracts the approach, tone, and methodology from those matches — not as a template to copy, but as a model to adapt from.

4
Critical Step

Recognize Novelty — What Makes This Ticket Unique

Even when a ticket looks familiar, it is never identical. This step forces the agent to identify the specific details that distinguish this ticket from historical matches — the customer's constraints, the error specifics, the timeline. This is where generic responses are prevented.

⚡ This is the step most AI tools skip. Without it, even a smart model drifts toward cookie-cutter answers. Forcing explicit novelty recognition is what keeps responses genuinely contextual.

5
Execution

Respond + Evolve — Anticipate the Next Reply

The response is crafted using your methodology, adapted for the novel specifics of this ticket. Crucially, the agent also anticipates how the ticket will evolve — what the customer is likely to say next — so the response already positions for the conversation, not just the question.

6
Learning

Learn — Update the Mental Model

After each exchange, the agent captures what this ticket taught: which diagnostic approach worked, what the customer's response revealed, whether the issue resolved. This learning is appended to the context archive — the 750-ticket baseline becomes 751, then 752.

7
Delivery

Compose & Send — Grounded in All 6 Steps Above

Only now is the response composed and posted to Zendesk. It's grounded in 750 real examples, filtered through your exact tone profile, adapted to this ticket's novel specifics, and built to advance the conversation toward resolution. Not a reply — a move.

🔁 When the customer replies, the loop restarts from Step 1 — but now with the updated context. Each cycle produces a more refined response than the last. The ticket gets smarter as it evolves.

In practice

What it looks like
in a real ticket

Ticket #100012 — VPN Not Connecting
Customer
I cannot connect to the VPN. It was working yesterday and now it just times out after 30 seconds.
Agent (Step 7 → First Response)
Hi, what exact error message do you see when it times out? And are you on Windows or Mac? Can you send a screenshot please?
Customer
Windows 10. It just says "connection timed out" — no other info. I also have antivirus running.
Agent (Step 7 → Loop 2)
That timeout on Windows with antivirus running is almost always the antivirus blocking the VPN client. Try disabling it temporarily and connecting — if that works, add the VPN client as an exception. Please let me know. Thanks!
Customer
That fixed it! How do I add the exception properly?
Agent (Step 7 → Loop 3 — Resolved)
In your antivirus settings we'll set a time to look for Exceptions or Exclusions and add the VPN client executable. That lets it run permanently without disabling protection.

Diagnosis-driven.
Not script-driven.

Notice what the agent doesn't do: it doesn't paste a 10-step troubleshooting checklist on the first reply. Instead it asks one targeted question, processes the answer, narrows the diagnosis, and resolves in three exchanges.

This mirrors how experienced support engineers actually work — and it's exactly what the 7-step framework learns from your ticket history.

  • ✓ First response: 1–2 targeted diagnostic questions (not a list of steps)
  • ✓ Second response: narrowed to the actual cause based on customer's answer
  • ✓ Resolution in the same conversation thread — no back-and-forth sprawl
  • ✓ Tone, closing style, and word count matched to your historical patterns
  • ✓ Ticket archived to context — next similar ticket handled even better
Run modes

Start cautious.
Scale with confidence.

The agent ships with two modes. Start in shadow to build trust, then flip to live when you're ready.

⚠️ Shadow Mode

Generate — Don't Send

The agent processes every ticket and generates responses, but queues them for your approval before anything touches Zendesk. Review, edit, approve, or reject each one.

  • All responses visible before posting
  • Zero risk of unwanted messages
  • Great for calibration and trust-building
  • Run with --mode shadow
🟢 Live Mode

Fully Autonomous

The agent monitors tickets, detects new customer replies, generates responses, and posts them directly to Zendesk — with no human in the loop.

  • Responds within seconds of a customer reply
  • Runs as a background daemon process
  • Configurable check interval (default 30s)
  • Run with --mode live
Get started

Up and running
in under 10 minutes.

.env 
# Your Zendesk credentials
ZENDESK_SUBDOMAIN=your-subdomain
ZENDESK_USER_EMAIL=you@yourcompany.com
ZENDESK_API_KEY=PASTE_YOUR_API_KEY_HERE

# Claude API (for AI responses)
ANTHROPIC_API_KEY=PASTE_YOUR_API_KEY_HERE
terminal 
# Shadow mode — review before sending
$ python3 auto_responder_daemon.py \
  --tickets YOUR_TICKET_ID \
  --mode shadow \
  --interval 30

# Live mode — fully autonomous
$ python3 auto_responder_daemon.py \
  --tickets YOUR_TICKET_ID \
  --mode live \
  --interval 30 \
  --duration 3600
1

Export your Zendesk tickets

Use the included zendesk_api_client.sh script to export your ticket history as JSON. Aim for 250+ tickets for best results.

2

Generate your taste profile

Open zendesk-taste-profile.html in a browser, drop in your ticket JSON, and generate a Markdown profile that captures your exact voice and patterns.

3

Set your credentials

Copy .env.example to .env and fill in your Zendesk subdomain, user email, API key, and Anthropic API key.

4

Start in shadow mode

Run the daemon in shadow mode and review a few generated responses. Once you're happy with the quality, switch to live mode.

5

Watch the context grow

Every ticket you handle is archived to trained_tickets.jsonl. The agent compounds with use — your 100th ticket is handled better than your first.
